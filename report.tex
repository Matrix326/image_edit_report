\begin{abstract}
图像编辑（Image Editing）任务以原始图像和自然语言编辑指令作为输入，输出修改后的图像。该任务不仅要求模型“改对”，还要求未被指令涉及的区域尽量保持不变。围绕这一目标，本报告首先对 InstructPix2Pix、Qwen-Image-Edit 与 EditAR 三类代表性基线进行统一复现，并在 MagicBrush 开发集样例上从指令执行正确性、内容保持度、SSIM/PSNR、BG-SSIM、Edit-Region Change 与 CLIP 相似度等维度进行评估。复现结果表明，Qwen-Image-Edit 的编辑语义最准确，但更容易发生背景或风格重绘；InstructPix2Pix 背景保持较好但复杂语义能力不足；EditAR 在该任务上整体可用性较弱。基于上述观察，本文在 Method Improvement 部分进一步提出面向 Qwen-Image-Edit 的监督目标改进：通过 GT LoRA 验证参数高效微调的有效性，通过 MTP（Masked Target Programming）显式构造背景保持的 clean target，并通过 MoE-Fusion Teacher 利用 P2P、Qwen 与 EditAR 等候选结果构造区域级 teacher 后蒸馏到单一 LoRA 模型中。实验表明，该改进能够在提升目标一致性的同时改善背景保持能力。
\end{abstract}
\keywords{图像编辑；指令编辑；MagicBrush；InstructPix2Pix；Qwen-Image-Edit；EditAR；LoRA；MTP；MoE-Fusion Teacher}

\section{Introduction}

图像编辑要求模型根据自然语言指令修改输入图像，同时保持非编辑区域的结构一致性。本文以 MagicBrush~\cite{magicbrush} 为基准，对三种不同范式的代表性模型进行系统复现与量化评估：(1) InstructPix2Pix（简记 P2P）——条件扩散式端到端指令编辑~\cite{instructpix2pix}；(2) Qwen-Image-Edit（简记 Qwen）——多模态大模型驱动的语义编辑；(3) EditAR——自回归离散 token 预测的统一条件生成~\cite{editar}。

贡献：(1) 在 MagicBrush 开发集 60 个样例上对三种基线进行系统复现，采用 SSIM、PSNR、BG-SSIM、Edit-Region Change 和 CLIP Score 共五类指标进行多维度评估；(2) 通过定量与可视化分析揭示三种范式在指令遵循、背景保持和编辑幅度之间的 trade-off；(3) 针对关键瓶颈提出改进方案与优化方向。

\section{Related Work}

\subsection{扩散模型与图像编辑}

\paragraph{扩散模型基础。}
扩散模型（Diffusion Models）通过定义前向加噪过程 $q(x_t|x_{t-1})$ 和反向去噪过程 $p_\theta(x_{t-1}|x_t)$ 来建模数据分布。DDPM~\cite{ddpm} 将前向过程定义为逐步添加高斯噪声的马尔可夫链，反向过程则训练一个 UNet 网络 $\epsilon_\theta(x_t,t)$ 预测所添加的噪声，通过最小化
\[
\mathbb{E}_{x_0,\epsilon,t}\left[\|\epsilon-\epsilon_\theta(x_t,t)\|^2\right]
\]
来优化。采样时从纯噪声 $x_T\sim\mathcal{N}(0,I)$ 出发，逐步去噪得到生成图像 $x_0$。

Latent Diffusion Model (LDM)~\cite{ldm} 将扩散过程从像素空间迁移到 VAE 压缩的潜空间。其架构包含三个核心组件：(1) 一个预训练的 VAE 编码器 $E$ 将图像压缩到潜表示 $z=E(x)$；(2) 一个在潜空间运行的 UNet 去噪网络 $\epsilon_\theta(z_t,t,\tau(c))$，通过交叉注意力层接收条件 $c$（文本、图像等）；(3) 一个 VAE 解码器 $D$ 将去噪后的潜表示恢复为图像。LDM 大幅降低了高分辨率合成的计算开销，成为 Stable Diffusion 等主流文生图模型的骨干架构。

\paragraph{可控图像生成。}
ControlNet~\cite{controlnet} 在冻结的预训练扩散模型旁添加一个可训练的“零卷积”副本编码器，将边缘图、深度图、姿态骨架等空间条件注入模型，实现了对生成结构的精细控制而无需微调原始权重。IP-Adapter 提出解耦的交叉注意力机制，将图像特征和文本特征分别通过独立的交叉注意力层注入 UNet，使得模型能够同时接受文本提示和图像提示（如风格参考图），在保持文本可控性的同时实现图像风格与内容迁移。

\paragraph{文本驱动的图像编辑。}
Prompt-to-Prompt（P2P）~\cite{p2p} 发现了扩散模型交叉注意力层的核心性质——注意力图编码了文本 token 与图像空间区域的对应关系。基于此，P2P 提出通过操控交叉注意力图来实现文本驱动的图像编辑：替换某个 token 的注意力图即可替换对应物体，对注意力图加权可控制编辑强度，注入新的注意力图可实现物体添加。该方法无需额外训练或 mask 标注。

\paragraph{InstructPix2Pix。}
InstructPix2Pix~\cite{instructpix2pix} 首次实现了端到端的指令式编辑。其训练流程为：(1) 使用 GPT-3 基于图像描述生成编辑指令和编辑后描述；(2) 使用 Prompt-to-Prompt 和 Stable Diffusion，以原始描述和编辑后描述分别生成编辑前后的图像对；(3) 以这些自动生成的图像对作为训练数据，训练一个以输入图像 $x$ 和编辑指令 $c$ 为双条件的扩散模型。推理时无需 DDIM 反演或 mask 标注，直接将输入图像和指令送入模型进行单次前向采样即可。本文选取其作为扩散式 baseline。

\subsection{多模态大模型与图像编辑}

随着大语言模型的成功，研究者开始将 LLM 的语义理解能力与图像生成模型结合，形成了多模态大模型驱动的图像编辑路线。

\paragraph{Qwen-Image。}
Qwen-Image 通过大规模多任务训练统一了文生图、图生图和图像重建三种能力。其核心设计为双编码器架构：(1) 语义编码器负责理解文本指令与图像的高级语义内容，提供编辑所需的语义指导；(2) 重建编码器专门保留输入图像的像素级细节信息（如纹理、边缘、颜色分布），以确保编辑后非目标区域的一致性。模型推理时，双编码器分别提取语义特征和重建特征，融合后作为条件注入扩散去噪过程，在“编辑得对”和“保持不变”之间取得平衡。

\paragraph{其他多模态编辑模型。}
EMU2 将多模态理解（图像描述、VQA）与生成（文生图、编辑）整合到单一自回归模型中，支持以对话方式进行多轮图像编辑。GILL 提出将 LLM 的文本嵌入通过一个可学习的映射网络转换到图像生成模型的输入空间，实现了 LLM 对 Stable Diffusion 的直接控制——用户用自然语言描述编辑需求，LLM 生成对应的条件嵌入，驱动扩散模型完成编辑。

本文选取 Qwen-Image-Edit 作为多模态大模型式 baseline，因其双编码器设计明确区分了语义理解与细节保持，代表性较强。

\subsection{自回归视觉生成与编辑}

与扩散模型的逐步去噪不同，自回归模型将图像生成建模为序列预测问题，形成了与扩散模型互补的另一条技术路线。

\paragraph{视觉 Token 化。}
VQGAN 奠定了自回归视觉生成的基础架构：首先使用 VQ-VAE 将图像编码为离散的视觉 token 序列（codebook indices），然后训练一个自回归 Transformer（如 GPT）以 next-token prediction 方式逐 token 生成图像序列，最后通过 VQ-VAE 解码器将 token 序列恢复为图像。该架构使标准 NLP 自回归范式可直接应用于视觉生成。

\paragraph{大规模自回归文生图。}
DALL-E 首次展示了自回归模型在大规模文生图任务上的强大能力，使用 120 亿参数的 Transformer 在 2.5 亿图文对上训练。Parti 将模型规模扩展至 200 亿参数，验证了自回归文生图遵循 scaling law——模型越大，生成质量越高、细节越丰富。LlamaGen 进一步证明了标准 Llama 架构无需视觉特化设计即可实现与扩散模型竞争的自回归图像生成。

\paragraph{EditAR。}
EditAR~\cite{editar} 将自回归范式扩展到统一条件生成框架。其核心思路是将输入图像、条件信号（如编辑指令、深度图、边缘图）和输出图像全部表示为统一的离散 token 序列，通过单一自回归 Transformer 处理多种条件生成任务。该框架的优势在于任务形式高度统一——图像编辑、深度估计、边缘检测、分割到图像等任务共用同一个模型，无需为每种任务设计独立的扩散结构。本文选取 EditAR 作为自回归式 baseline，以评估该范式在指令式图像编辑中的实际表现。

\subsection{评测基准}

MagicBrush~\cite{magicbrush} 是首个大规模人工标注的指令式图像编辑数据集，约 10,000 条样例，每条配备输入图像、编辑指令、人工编辑的目标图像和编辑区域 mask，覆盖背景替换、物体增删、属性修改、姿态变化等类别，其多轮编辑子集支持序列化编辑场景。本文使用其开发集 60 个样例作为评测基准。

\section{Baseline Reproduction}

\subsection{评测指标与实验设置}

\subsubsection{评测指标}

设输入图像为 $I\in\mathbb{R}^{H\times W\times 3}$，模型输出为 $O$，人工目标图像为 $G$，编辑区域二值 mask 为 $M\in\{0,1\}^{H\times W}$（$M_{ij}=1$ 表示像素 $(i,j)$ 属于编辑区域）。采用以下五类指标：
\begin{itemize}
  \item \textbf{SSIM / PSNR (Target, Output)：}衡量 $O$ 与 $G$ 的结构相似度和像素级保真度，反映编辑输出与人工目标的整体一致性。SSIM 从亮度、对比度、结构三个维度综合评估：
\[
\operatorname{SSIM}(x,y)=
\frac{(2\mu_x\mu_y+C_1)(2\sigma_{xy}+C_2)}
{(\mu_x^2+\mu_y^2+C_1)(\sigma_x^2+\sigma_y^2+C_2)}.
\]
  \item \textbf{BG-SSIM：}仅在非编辑区域（$M_{ij}=0$）计算 SSIM，衡量模型对背景的保持能力。该指标直接反映模型是否在无需修改的区域引入了不必要的变化。
  \item \textbf{SSIM / PSNR (Input, Output)：}衡量 $O$ 与 $I$ 的整体相似度——值越高说明模型对原图的改动越小（越保守）。需结合编辑指令判断：保守不一定好（可能编辑不足）。
  \item \textbf{Edit-Region Change：}编辑区域内的平均像素绝对变化量，需结合指令语义解读——过小表示编辑不足，过大可能表示过度修改。
  \item \textbf{CLIP Score (Target, Output)：}使用 CLIP-ViT-L/14 提取 $O$ 与 $G$ 的图像特征，计算余弦相似度，衡量语义级别的编辑一致性。这是对像素级指标的重要补充。
\end{itemize}

\subsubsection{实验设置}

三种 baseline 均使用官方发布的预训练权重，不做任何微调。P2P 基于 Stable Diffusion v1.5 骨干，扩散步数 $T=100$，分类器无关引导强度 $\omega=7.5$，图像引导强度和文本引导强度均设为 1.5。Qwen-Image-Edit 使用 Qwen-Image 官方推理配置，输入分辨率统一缩放至 $512\times512$，采用默认的双编码器融合权重。EditAR 使用 VAR-d30 自回归模型，top-$k=100$、top-$p=0.9$，图像 token 序列长度 $N=1024$。全部实验在单卡 NVIDIA A100-80GB GPU 上完成，推理时关闭梯度计算以最大化显存利用。

\subsection{InstructPix2Pix（扩散式）}

\subsubsection{技术原理}

InstructPix2Pix~\cite{instructpix2pix} 是基于 Stable Diffusion 的条件扩散模型。其核心创新在于训练数据的自动构造和双条件注入机制。

\paragraph{训练数据构造流水线。}
训练过程无需任何人工标注：
\begin{itemize}
  \item 步骤 1：使用 BLIP 等模型为真实图像生成文本描述；
  \item 步骤 2：使用 GPT-3 基于描述生成编辑指令 $c$ 和编辑后描述 $t_{\mathrm{edit}}$；
  \item 步骤 3：使用 Prompt-to-Prompt 技术，以原始描述 $t_{\mathrm{orig}}$ 和 $t_{\mathrm{edit}}$ 分别通过 Stable Diffusion 生成编辑前后的图像对 $(x_{\mathrm{before}},x_{\mathrm{after}})$。
\end{itemize}

\paragraph{模型架构。}
模型以输入图像 $x$ 和编辑指令 $c$ 为双条件。输入图像通过 VAE 编码器得到潜表示，与噪声拼接后送入 UNet。编辑指令经 CLIP 文本编码器编码后通过交叉注意力注入 UNet。模型学习预测添加到潜表示上的噪声，去噪后经 VAE 解码器得到编辑后图像 $y$。形式化地：
\[
p_\theta(y\mid x,c)=
\int p_\theta(z_T)\prod_{t=1}^{T}
p_\theta(z_{t-1}\mid z_t,E(x),\tau(c))\,dz_{1:T},
\qquad y=D(z_0).
\]
其中 $E$ 为 VAE 编码器，$D$ 为解码器，$\tau$ 为文本编码器。推理时采用分类器无关引导（CFG），同时以空图像和空文本为负条件，通过两个引导尺度分别控制图像条件强度和文本条件强度。

\subsubsection{典型案例}

P2P 处于 Qwen 与 EditAR 之间的中间态：不如 Qwen 精准，但比 EditAR 的语义理解强得多。在简单属性编辑上表现极佳（dev\_05 SSIM 0.9919），且背景保持为三种方法最优（BG-SSIM 0.7952）。但在复杂空间推理（dev\_31 “bird on zebra head”，SSIM 0.412）和局部替换（dev\_37 “mirror$\rightarrow$window”，SSIM 0.107，全数据集 P2P 最差）中容易全局失控。60 样例平均 SSIM 0.7228，处于不上不下的位置。

\subsection{Qwen-Image-Edit（多模态大模型式）}

\subsubsection{技术原理}

Qwen-Image-Edit 基于 Qwen-Image 多模态基础模型。Qwen-Image 通过大规模多任务训练（文生图、图生图、图像重建）在统一框架内同时优化生成质量与编辑一致性。

\paragraph{双编码器架构。}
Qwen-Image 的核心创新在于解耦了语义理解和细节保持：
\begin{itemize}
  \item \textbf{语义编码器：}基于 Qwen-VL 视觉语言模型，负责理解输入图像的高级语义内容（物体类别、场景布局、人物身份等）以及解析编辑指令的意图。它将图像和文本指令映射到共享的语义特征空间。
  \item \textbf{重建编码器：}专注于保留输入图像的像素级细节——纹理、边缘、颜色分布、光照条件等低层视觉特征。该编码器确保模型在语义层面进行编辑时不会丢失输入图像的视觉一致性。
\end{itemize}

\paragraph{推理流程。}
给定输入图像 $I$ 和编辑指令 $T$：(1) 语义编码器从 $I$ 和 $T$ 中提取语义特征 $f_{\mathrm{sem}}$；(2) 重建编码器从 $I$ 中提取重建特征 $f_{\mathrm{rec}}$（保持细节信息）；(3) $f_{\mathrm{sem}}$ 和 $f_{\mathrm{rec}}$ 通过可学习的融合权重拼接后，作为条件注入扩散模型的 UNet 交叉注意力层；(4) 扩散过程在条件约束下生成编辑后图像。

\paragraph{关键特性。}
双编码器带来的优势是：当编辑指令语义明确时，语义编码器主导编辑方向；当编辑指令不涉及某些区域时，重建编码器自动保持这些区域的像素一致性。但缺点在于：当重建编码器权重过大时，模型倾向于“过度保守”——即使在需要修改的区域也尽量保持原样，导致编辑幅度不足；而当语义编码器主导时，也可能引发全局性的风格重绘——模型不仅修改了指定区域，还将整个图像的色调、光照、纹理都向训练数据中见过的风格偏移。

\subsubsection{典型案例}

Qwen 是三款 baseline 中编辑最精准的：dev\_07（SSIM 0.8063）是唯一成功生成可辨识光剑的模型，主观效果可接受。但偶发的风格级重绘是致命缺陷——dev\_46（“white socks”）BG-SSIM 仅 0.237，几乎重绘了整幅图像；dev\_58（“turn hair white”）SSIM 0.2308 为全数据集最差。60 样例平均 SSIM 0.7804（最高）、BG-SSIM 0.6910（最低），验证了“最准但背景最弱”的特征。

\subsection{EditAR（自回归式）}

\subsubsection{技术原理}

EditAR~\cite{editar} 采用自回归范式进行图像编辑，与扩散模型从噪声逐步去噪不同，自回归模型将图像视为离散 token 序列，以 next-token prediction 方式逐 token 生成。

\paragraph{统一 Token 化。}
EditAR 的核心设计是将所有模态统一表示为离散 token：
\begin{itemize}
  \item \textbf{图像 Token 化：}使用预训练的 VQ-VAE 将输入图像 $I$ 编码为 $N$ 个离散 token $\{v_1,v_2,\ldots,v_N\}$，每个 token 来自大小为 $K$ 的 codebook。同样将目标输出图像表示为 token 序列。
  \item \textbf{条件 Token 化：}编辑指令 $c$ 经文本 tokenizer 编码为文本 token 序列。对于非文本条件（如深度图、边缘图），同样使用对应的 VQ-VAE 编码为 token。
\end{itemize}

\paragraph{自回归生成。}
所有 token 拼接为一个统一的序列，送入自回归 Transformer 进行 next-token prediction 训练。Transformer 学习条件分布：
\[
p(\{y_i\}_{i=1}^{N}\mid I,c)
=
\prod_{i=1}^{N}p(y_i\mid y_{<i},\operatorname{tok}(I),\operatorname{tok}(c)).
\]
推理时，模型从条件 token（输入图像 + 编辑指令）开始，逐 token 自回归生成输出图像的 token 序列，最后通过 VQ-VAE 解码器将 token 恢复为图像。

\paragraph{统一条件生成。}
EditAR 的关键优势在于单一框架处理多种任务：只需改变条件类型（编辑指令、深度图、边缘图、分割图），同一模型即可完成图像编辑、深度估计、边缘检测到图像、分割到图像等不同任务，无需为每种任务设计专门的架构或训练流程。

\section{Method Improvement}

\subsection{Qwen-Image-Edit 系列改进方案技术报告}

\noindent\textbf{作者：}王溢韬 \qquad \textbf{时间：}2026 年春季学期

\paragraph{摘要。}
本文围绕 Qwen-Image-Edit 系列图像编辑模型展开。我们首先采用更新的 Qwen-Image-Edit-2511 作为基座模型，测试其在 MagicBrush 指令图像编辑任务上的基础能力；随后在 MagicBrush 训练集上进行 LoRA 微调。实验表明，直接使用数据集真值进行 LoRA 微调可以显著提升特定任务性能，但仍存在编辑位置不够准确和背景保持不足的问题。针对这些问题，最终提出两条改进路线：一是 \textbf{MTP}，只使用 mask-preserved target 进行 LoRA 训练，二是 \textbf{MoE-Fusion Teacher}，利用 Pix2Pix、Qwen-Image-Edit、EditAR 进行多专家候选，通过 target/mask 监督构造区域级 teacher，再蒸馏到 Qwen2511 LoRA。MagicBrush 的 528 条测试样例显示，GT LoRA 将 Target--Output SSIM 从 0.450 提升到 0.696；MTP 进一步提升到 0.740；MoE Teacher LoRA 达到 0.763，并取得最高 BG-SSIM。

\subsubsection{任务、数据与评测}

指令图像编辑的输入为原图 $I$ 和文本指令 $p$，输出为编辑图 $O$。监督数据来自 MagicBrush，每个样本包含：
\[
(I,p,G,M),
\]
其中 $G$ 是人工目标图像，$M\in\{0,1\}^{H\times W}$ 是编辑区域 mask。本实验使用的划分为：
\begin{itemize}
  \item \textbf{训练集：}MagicBrush train，共 8807 条样本。
  \item \textbf{测试集：}MagicBrush dev，共 528 条样本。
  \item \textbf{图像尺寸：}训练与推理使用 Qwen-Image-Edit-2511 图像编辑接口，最大像素数为 $512^2=262144$。
\end{itemize}

\[
O=f_{\theta}(I,p),
\]

\[
\texttt{edit\_image}=[I],\quad
\texttt{prompt}=p,\quad
\texttt{image}=Y,
\]

\bar G, & \text{MTP LoRA},\\
T_{\mathrm{moe}}, & \text{MoE Teacher LoRA}.
\end{cases}
\]
因此三条路线推理接口相同：
\[
\texttt{input image}+\texttt{instruction}
\longrightarrow
\texttt{edited image}.
\]

\subsubsection{Qwen2511 Base 与 GT LoRA}

\paragraph{Base 模型。}
首先测试原始 Qwen-Image-Edit-2511，不加载任何 LoRA：
\[
O=f_{\theta_0}(I,p).
\]
Qwen2511 Base 具有较强语言理解和通用编辑先验，实际效果发现对大部分指令都能基本完成；但它没有针对 MagicBrush 的目标进行适配，Target--Output SSIM 只有 0.450，TO-PSNR 为 14.29。在视觉效果上表现为编辑位置不够准确，且会影响背景。

\paragraph{基于 MagicBrush 数据集的 LoRA 微调。}
GT LoRA 的目标是将 Qwen2511 适配到 MagicBrush 的监督分布。

LoRA 冻结 Qwen2511 的原始 DiT 权重 $W_0$，在目标线性层上增加低秩更新：
\[
W'=W_0+\Delta W,\qquad
\Delta W=\frac{\alpha}{r}BA,
\]
其中 $A\in\mathbb{R}^{r\times d}$，$B\in\mathbb{R}^{k\times r}$，$r$ 是 LoRA rank，$\alpha$ 是缩放系数。训练时只更新 $A,B$：
\[
\theta^\star_{\text{LoRA}}
=
\arg\min_{\theta_{\text{LoRA}}}
\mathbb{E}_{(I,p,G,M)}
\left[
\mathcal{L}(f_{\theta_0,\theta_{\text{LoRA}}}(I,p),G,M)
\right].
\]
LoRA 注入 Qwen2511 DiT 的 attention 与 modulation/MLP 层，GT LoRA 设置为 rank 32，学习率 $1\times10^{-4}$，完整训练 1 个 epoch，4 张 A100 并行。

\paragraph{LoRA 训练与推理完整流程。}
Qwen-Image-Edit-2511 的图像编辑管线可以抽象为四个部分：文本编码器、图像 VAE、扩散 DiT 和调度器。训练时，输入图 $I$ 作为条件图像，目标图 $Y$ 作为监督图像。具体流程如下。

\textbf{训练输入。}
每个样本包含：
\[
(I,p,Y,M,w_s),
\]
其中 $Y$ 是目标图，$M$ 是训练期 mask，$w_s$ 是样本权重。首先用 VAE 编码目标图与条件图：
\[
z_Y=\operatorname{Enc}_{\mathrm{vae}}(Y),\qquad
z_I=\operatorname{Enc}_{\mathrm{vae}}(I).
\]
文本指令由 Qwen text encoder 编码：
\[
h_p=\operatorname{Enc}_{\mathrm{text}}(p).
\]

\textbf{加噪与训练目标。}
对目标隐变量 $z_Y$ 采样时间步 $t$ 与噪声 $\epsilon$，得到训练隐变量：
\[
z_t=a(t)z_Y+b(t)\epsilon.
\]
调度器根据流匹配定义训练目标 $u_t$。在实现中，$u_t$ 含义为从 $z_t$ 回到 $z_Y$ 所需的速度向量。

\textbf{条件预测。}
DiT 接收加噪隐变量、文本条件和图像条件：
\[
\hat u_t
=
D_{\theta_0+\Delta\theta}(z_t,t,h_p,z_I).
\]
其中 $\theta_0$ 是冻结的 Qwen2511 base 参数，$\Delta\theta$ 是所有 LoRA adapter 参数。

\textbf{优化目标。}
每一步只更新 LoRA 参数：
\[
\Delta\theta^\star
=
\arg\min_{\Delta\theta}
\mathbb{E}
\left[
\mathcal{L}_{\mathrm{region}}(\hat u_t,u_t,M,w_s)
\right].
\]
优化完成后保存为 \texttt{.safetensors} 格式的 LoRA 权重。推理时，将 LoRA 融合进 Qwen2511 DiT，然后执行标准反向扩散：
\[
O=\operatorname{Decode}_{\mathrm{vae}}\left(
\operatorname{Denoise}_{\theta_0+\Delta\theta}(I,p)
\right).
\]

\paragraph{Mask-aware 流匹配损失。}
Qwen2511 的监督训练采用流匹配。设目标图像 $Y$ 的隐变量为 $z_0$，加噪隐变量为：
\[
z_t=a(t)z_0+b(t)\epsilon,\qquad \epsilon\sim\mathcal{N}(0,I),
\]
模型预测速度场 $v_\theta(z_t,t,I,p)$，真实训练目标为 $v^\star(z_0,\epsilon,t)$。逐元素误差为：
\[
E=\left\|v_\theta(z_t,t,I,p)-v^\star\right\|_2^2.
\]

直接对 $E$ 做全图平均会让大面积背景主导训练。为此，我们将编辑 mask resize 到隐变量尺度，记为 $\hat M$，背景为 $\hat B=1-\hat M$，并做分区归一化：
\[
\mathcal{L}_{\mathrm{edit}}=\frac{\sum \hat M E}{\sum \hat M+\varepsilon},\qquad
\mathcal{L}_{\mathrm{bg}}=\frac{\sum \hat B E}{\sum \hat B+\varepsilon}.
\]
最终损失为：
\[
\mathcal{L}_{\mathrm{sft}}
=
w_s\cdot
\frac{\lambda_e\mathcal{L}_{\mathrm{edit}}+\lambda_b\mathcal{L}_{\mathrm{bg}}}
{\lambda_e+\lambda_b}.
\]
其中 $w_s$ 是样本权重。GT LoRA 默认使用 $\lambda_e=1.5,\lambda_b=0.5$。这使编辑区域的梯度不会被背景像素数量淹没。

\paragraph{GT LoRA 的效果与问题。}
GT LoRA 将 TO-SSIM 从 0.450 提升到 0.696，TO-PSNR 从 14.29 提升到 18.24，说明 LoRA 微调对 Qwen 大模型进行特定任务适配非常有效。

但 GT LoRA 仍有两个问题：
\begin{enumerate}
  \item \textbf{位置理解不足。}模型能理解 prompt，但不一定知道该在图中哪个精确区域修改。\texttt{371643\_1} 中能理解香蕉语义，但替换范围和布局仍不稳定。
  \item \textbf{背景保持不足。}在 \texttt{45991\_1}，模型改变了原本茶杯的位置。
\end{enumerate}

\subsubsection{改进一：MTP LoRA}

\paragraph{核心思想。}
MTP 的全称为 Masked Target Programming。它不直接让模型拟合整张 $G$，而是构造一个更适合局部编辑训练的 clean target：
\[
\bar G=\tilde M\odot G+(1-\tilde M)\odot I.
\]
其中 $\tilde M\in[0,1]^{H\times W}$ 是 soft edit mask。该目标的含义是：
\begin{itemize}
  \item 编辑区域使用目标图 $G$，保证模型学习具体编辑内容；
  \item 背景区域使用原图 $I$，显式告诉模型不要修改无关区域；
  \item mask 边界做了柔和处理，避免硬拼接带来的边界不自然。
\end{itemize}

\paragraph{Mask 生成与选择。}
虽然数据集本身具有 mask，但由于 MagicBrush 是人工数据，实际观察发现并不总是那么合适的，为此构造多个候选：
\[
\mathcal{M}=
\{M_{\mathrm{diff}}, M_{\mathrm{dataset}}, 1-M_{\mathrm{dataset}}\}.
\]
其中 diff mask 来自 source-target 差异：
\[
D(i,j)=\frac{1}{3}\sum_c |I(i,j,c)-G(i,j,c)|.
\]
对 $D$ 使用固定阈值得到二值 mask，并经过移除杂质、破洞等清理。

每个候选 mask 的评分由四部分组成：
\[
S_M(M)
=
\underbrace{\mu(D\mid M=1)-\mu(D\mid M=0)}_{\text{区域区分度}}
+\beta\underbrace{\frac{\sum M D}{\sum D+\varepsilon}}_{\text{差异覆盖率}}
-\gamma A_{\mathrm{broad}}(M)
-\eta A_{\mathrm{tiny}}(M).
\]
选择评分最高者对应的 mask。

如果指令属于全局编辑，或 mask 面积超过阈值，则使用全图编辑，令 $\tilde M=\mathbf{1}$。

统计结果：8807 个训练样本，267 条 no-op 保持样本；36 条 global edit；8774 条选择 source-target diff mask，25 条选择 dataset mask，8 条选择反相 dataset mask；平均编辑 mask 面积为 0.0959。

\paragraph{MTP 完整算法流程。}
MTP 将训练目标重写。它把原始监督目标 $G$ 改写成更适合局部编辑学习的 $\bar G$，再用 $\bar G$ 对 Qwen2511 做 LoRA SFT。

\textbf{第一步：构造 source-target 差异图。}
将 $G$ resize 到 $I$ 的大小，计算逐像素平均差异：
\[
D(i,j)=\frac{1}{3}\sum_{c=1}^{3}\left|I(i,j,c)-G(i,j,c)\right|.
\]
如果 $D$ 的 Otsu 阈值为 $\tau_{\mathrm{otsu}}$，则初始 diff mask 为：
\[
M_{\mathrm{diff}}^{0}(i,j)=\mathbf{1}[D(i,j)>\tau_{\mathrm{otsu}}].
\]
若 Otsu 得到的区域过小，则使用固定阈值 fallback：
\[
M_{\mathrm{diff}}^{0}(i,j)=\mathbf{1}[D(i,j)>\tau_{\mathrm{fix}}].
\]

\textbf{第二步：候选 mask 清理。}
对每个候选 mask 执行同一套形态学清理：
\[
\operatorname{Clean}(M)
=
\operatorname{Dilate}\left(
\operatorname{Close}\left(
\operatorname{FillHoles}\left(
\operatorname{RemoveSmall}(M)
\right)\right)\right).
\]
最终候选集合为：
\[
\mathcal{M}
=
\left\{
\operatorname{Clean}(M_{\mathrm{diff}}^{0}),
\operatorname{Clean}(M_{\mathrm{data}}),
\operatorname{Clean}(1-M_{\mathrm{data}})
\right\}.
\]
加入 $1-M_{\mathrm{data}}$ 是因为部分数据集 mask 可能记录的是保留区域而不是编辑区域，反相候选可以自动修正极性错误。

\textbf{第三步：候选 mask 打分与选择。}
对候选 mask $M$，定义面积、mask 内外差异和覆盖率：
\[
a(M)=\frac{1}{HW}\sum_{i,j}M_{ij},
\]
\[
d_{\mathrm{in}}(M)=\frac{\sum M\odot D}{\sum M+\varepsilon},\qquad
d_{\mathrm{out}}(M)=\frac{\sum (1-M)\odot D}{\sum (1-M)+\varepsilon},
\]
\[
c(M)=\frac{\sum M\odot D}{\sum D+\varepsilon}.
\]
MTP 希望 mask 覆盖真正变化区域，但不能过大也不能过小，因此采用：
\[
S_M(M)=
d_{\mathrm{in}}(M)-d_{\mathrm{out}}(M)
+\beta c(M)
-\gamma\max(0,a(M)-a_{\mathrm{broad}})
-\eta\max(0,a_{\mathrm{tiny}}-a(M)).
\]
如果指令属于全局编辑，例如风格、光照、背景整体改变，则允许大面积 mask；否则大面积 mask 会被惩罚。最终选择：
\[
M^\star=\arg\max_{M\in\mathcal{M}}S_M(M).
\]

\textbf{第四步：从 hard mask 到 soft mask。}
局部编辑不能使用硬边界直接拼接。MTP 先对 $M^\star$ 膨胀，再做高斯 feather：
\[
M_d=\operatorname{Dilate}(M^\star,r_d),
\qquad
\tilde M=\operatorname{clip}\left(\mathcal{G}_{\sigma}*M_d,0,1\right).
\]
边界 mask 用于额外约束接缝区域：
\[
M_{\mathrm{bd}}
=
\operatorname{Dilate}(M^\star,r_{\mathrm{bd}})
-
\operatorname{Erode}(M^\star,r_{\mathrm{bd}}).
\]

\textbf{第五步：构造 clean target。}
对局部编辑样本，监督目标为：
\[
\bar G=\tilde M\odot G+(1-\tilde M)\odot I.
\]
对全局编辑样本，令 $\tilde M=\mathbf{1}$，于是 $\bar G=G$。保证 MTP 不会错误地把全局风格任务压成局部替换任务。

\textbf{第六步：加入 no-op 保持样本。}
为了让模型学会“没有修改要求时不要改图”，每隔固定 stride 增加一条保持样本：
\[
I+\text{``Do not change the image''}\rightarrow I.
\]
这类样本的 mask 为全 0，样本权重较低，不主导训练，只作为保持能力正则。

\textbf{第七步：训练 Qwen2511 LoRA。}
使用与 Qwen2511 LoRA 同样的训练流程，但把目标 $Y$ 设为 $\bar G$。

\paragraph{MTP LoRA 损失。}
MTP-PhaseA 使用 clean target $\bar G$ 的隐变量作为流匹配监督，同时引入编辑区、背景区和边界区三项区域归一化损失：
\[
\mathcal{L}_{\mathrm{mtp}}
=
\frac{
\lambda_e\mathcal{L}_{\mathrm{edit}}
+\lambda_b\mathcal{L}_{\mathrm{bg}}
+\lambda_{\mathrm{bd}}\mathcal{L}_{\mathrm{bd}}}
{\lambda_e+\lambda_b+\lambda_{\mathrm{bd}}}.
\]
其中：
\[
\mathcal{L}_{\mathrm{bd}}
=
\frac{\sum \hat M_{\mathrm{bd}} E}{\sum \hat M_{\mathrm{bd}}+\varepsilon}.
\]
最终采用的 PhaseA 设置为：
\[
\lambda_e=4.0,\quad
\lambda_b=0.3,\quad
\lambda_{\mathrm{bd}}=0.15.
\]
LoRA rank 为 16，学习率为 $5\times10^{-5}$，训练 1 个 epoch。

\paragraph{MTP LoRA 结果。}
MTP LoRA 的 TO-SSIM 为 0.740，高于 GT LoRA 的 0.696；BG-SSIM 为 0.828，高于 GT LoRA 的 0.821；Edit Change 为 0.172，远高于 GT LoRA 的 0.043。这说明 MTP LoRA 同时提升了目标相似度、背景保持和编辑发生强度，模型不仅更敢于去改动，同时也改的更正确。

\subsubsection{改进二：MoE-Fusion Teacher}

\paragraph{核心思想。}
对于 MagicBrush 这种给出了原始图和目标图的数据集来说，用 mask 作为监督信号是可以的，但是 mask 也有局限，有时对于风格的修改，比如“把夏天改成秋天”，这种指令，是单独 mask 无法准确反映的，而且对于同一指令，不同区域可能需要不同的编辑策略，而且结果也不是唯一的，目标图只能作为一种参考，为此想到，能否构建一个教师，能够交给模型一个修改方向而非仅仅是答案呢？为此希望在基线的三个模型基础上，发挥每个模型的长处，善于生成正确图像的负责生成正确图像，善于保持背景的负责背景的维持。MoE-Fusion Teacher 的目标是利用多个专家在不同区域的优势构造训练期 teacher，作为监督微调的目标。

专家候选集合为：
\[
\mathcal{C}=\{C_{\mathrm{pix2pix}},C_{\mathrm{qwen}},C_{\mathrm{editar}}\}.
\]
由于实际结果发现，Qwen 性能非常优异，实际实现还为 Qwen 候选生成 conservative/aggressive 虚拟变体，以覆盖不同编辑强度：
\[
C_{\mathrm{qwen\mbox{-}cons}}
=
\operatorname{clip}\left(I+s_c(C_{\mathrm{qwen}}-I),0,1\right),
\qquad s_c=0.65,
\]
\[
C_{\mathrm{qwen\mbox{-}aggr}}
=
\operatorname{clip}\left(I+s_a(C_{\mathrm{qwen}}-I),0,1\right),
\qquad s_a=1.25.
\]
因此 MoE 实际路由的专家族为：
\[
\mathcal{E}
=
\{\text{pix2pix},\text{qwen},\text{qwen-cons},\text{qwen-aggr},\text{editar}\}.
\]

\paragraph{专家评分。}
对每个候选 $C$，在编辑区、背景区、方向一致性和边界自然度上评分。编辑区目标相似：
\[
E_{\mathrm{edit}}(C)=\operatorname{MSE}(C,G;M)
+\alpha_s\left(1-\operatorname{SSIM}(C,G;M)\right).
\]
背景保持：
\[
E_{\mathrm{bg}}(C)=\operatorname{MSE}(C,I;1-M)
+\alpha_b\left(1-\operatorname{SSIM}(C,I;1-M)\right).
\]
编辑方向一致性比较候选和 target 相对 source 的变化方向：
\[
D_{\mathrm{dir}}(C)=1-\cos\left((C-I)\odot M,(G-I)\odot M\right).
\]
边界惩罚衡量 mask 边界附近的颜色和梯度跳变：
\[
B(C)=
\rho\|C-I\|_{1,M_{\mathrm{bd}}}
+(1-\rho)\|\nabla C-\nabla I\|_{1,M_{\mathrm{bd}}}.
\]
综合得分越低越好：
\[
S(C)=
\lambda_e E_{\mathrm{edit}}(C)
+\lambda_b E_{\mathrm{bg}}(C)
+\lambda_d D_{\mathrm{dir}}(C)
+\lambda_{\mathrm{bd}}B(C)
+\lambda_f\operatorname{MSE}(C,G).
\]
整图评分用于候选质量统计；局部评分 $S_k(C)$ 在每个连通区域 $M_k$ 上重新计算 edit MSE、edit SSIM、方向一致性、边界惩罚，并加入一个不修改惩罚：
\[
P_{\mathrm{no\mbox{-}change}}(C,M_k)
=
\max\left(0,\tau_{\mathrm{chg}}-\operatorname{MSE}(C,I;M_k)\right).
\]
这样可以避免某个专家因为几乎不改图而在像素误差上看起来很小，但实际上没有完成编辑。

当最优专家和第二名专家的分数分别为 $S_1,S_2$ 时，区域置信度定义为：
\[
\operatorname{conf}
=
\operatorname{clip}\left(
0.55+0.45\tanh
\frac{S_2-S_1}{0.25\max(|S_1|,|S_2|)+\varepsilon},
0.55,1.0
\right).
\]
置信度越高，说明最优专家明显优于其他专家；置信度越低，说明多个专家质量接近，需要更柔和的融合。

\paragraph{区域级路由与融合。}
MoE 不做整图选择，而是把编辑 mask 分解为连通区域：
\[
M=\bigcup_{k=1}^{K}M_k.
\]
每个区域独立选择专家：
\[
e_k^\star
=
\arg\min_{e\in\mathcal{E}} S_k(C_e).
\]
如果 $\operatorname{conf}_k$ 高于硬路由阈值，则直接使用该专家；如果置信度较低，则在 top-$K$ 专家之间做软路由：
\[
\pi_{k,e}
=
\frac{\exp(-S_k(C_e)/\tau)}
{\sum_{e'\in\operatorname{TopK}(k)}\exp(-S_k(C_{e'})/\tau)},
\]
\[
C_k^{\mathrm{route}}=\sum_{e\in\operatorname{TopK}(k)}\pi_{k,e}C_e.
\]

背景区域默认选择 source 或背景保持得分最好的专家：
\[
e_{\mathrm{bg}}^\star
=
\arg\min_{e\in\mathcal{E}\cup\{\text{source}\}}E_{\mathrm{bg}}(C_e).
\]
对局部编辑样本，默认把 source 作为背景专家，即：
\[
C_{\mathrm{bg}}=I.
\]

初始融合图为：
\[
T_0
=
(1-\tilde M)\odot C_{e_{\mathrm{bg}}^\star}
+\sum_{k=1}^{K}\tilde M_k\odot C_{e_k^\star}.
\]
其中 $\tilde M_k$ 是每个连通区域的特征 mask。之后执行颜色协调化和拉普拉斯融合，使边界更自然：
\[
H(C_k)
=
\operatorname{clip}
\left(C_k+\rho_h(\mu_{\mathrm{bg}}^{\mathrm{bd}}-\mu_k^{\mathrm{bd}}),0,1\right),
\]
其中 $\mu_{\mathrm{bg}}^{\mathrm{bd}}$ 和 $\mu_k^{\mathrm{bd}}$ 分别是边界带上背景和候选区域的 RGB 均值，$\rho_h$ 是协调化强度。再将前景与背景做金字塔融合：
\[
T_{\mathrm{moe}}=\operatorname{Blend}_{\mathrm{lap}}(\operatorname{Harmonize}(T_0,I,M)).
\]

\paragraph{MoE-Fusion Teacher 完整算法流程。}
\textbf{输入与训练期可用信息。}
MoE 的输入为：
\[
(I,p,G,M,\mathcal{C}),
\]
其中 $\mathcal{C}$ 是 Pix2Pix、Qwen-Image-Edit、EditAR 生成的候选图集合。

\textbf{第一步：专家候选生成。}
三个真实专家分别产生：
\[
C_{\mathrm{pix2pix}}=f_{\mathrm{pix2pix}}(I,p),\quad
C_{\mathrm{qwen}}=f_{\mathrm{qwen}}(I,p),\quad
C_{\mathrm{editar}}=f_{\mathrm{editar}}(I,p).
\]

\textbf{第二步：全局候选评分。}
对每个 $C_e$ 计算 $E_{\mathrm{edit}}$、$E_{\mathrm{bg}}$、$D_{\mathrm{dir}}$、$B$ 和 $S(C_e)$，分数用于：
\begin{itemize}
  \item 判断哪个专家在编辑区最接近 target；
  \item 判断哪个专家或 source 背景保持最好；
  \item 为最终 teacher confidence 和样本权重提供依据；
  \item 为失败样例分析提供可解释证据。
\end{itemize}

\textbf{第三步：编辑区域连通分量路由。}
把 $M$ 划分为 $K$ 个连通分量 $\{M_k\}_{k=1}^{K}$。对每个分量重新计算局部分数 $S_k(C_e)$：
\[
\operatorname{rank}_k
=
\operatorname{argsort}_{e\in\mathcal{E}}S_k(C_e).
\]

\textbf{第四步：背景路由。}
对局部编辑，背景优先使用 source：
\[
C_{\mathrm{bg}}=I.
\]
对全局编辑或大面积 mask，背景候选也参与排序，避免强制 source 背景破坏全局风格编辑。

\textbf{第五步：区域融合。}
每个区域生成对应特征 mask $\tilde M_k$。如果硬路由，则：
\[
F_k=C_{e_k^\star};
\]
如果软路由，则：
\[
F_k=\sum_e \pi_{k,e}C_e.
\]
所有区域叠加到背景上：
\[
T_0=(1-\tilde M)\odot C_{\mathrm{bg}}
+\sum_{k=1}^{K}\tilde M_k\odot F_k.
\]
随后做颜色协调化和拉普拉斯融合，得到 $T_{\mathrm{moe}}$。

\textbf{第六步：teacher 质量保护。}
重新评价融合 teacher 的综合分数 $S(T_{\mathrm{moe}})$。若
\[
S(T_{\mathrm{moe}})> \min_e S(C_e)+\delta,
\]
说明融合图比最佳单专家更差，则回退到最佳单专家。

\paragraph{MoE Teacher LoRA 蒸馏。}
MoE teacher 只用于训练目标。学生仍然是 Qwen2511 source-only LoRA：样本权重由 teacher conf 调节：
\[
w_s=\operatorname{clip}(w_{\min}+\operatorname{conf}(T_{\mathrm{moe}})(w_{\max}-w_{\min}),w_{\min},w_{\max}).
\]
训练损失仍采用 mask-aware 流匹配：
\[
\mathcal{L}_{\mathrm{moe}}=w_s
\frac{\lambda_e\mathcal{L}_{\mathrm{edit}}^{\mathrm{moe}}
+\lambda_b\mathcal{L}_{\mathrm{bg}}^{\mathrm{moe}}}
{\lambda_e+\lambda_b}.
\]
其中 $\mathcal{L}_{\mathrm{edit}}^{\mathrm{moe}}$ 与 $\mathcal{L}_{\mathrm{bg}}^{\mathrm{moe}}$ 的定义和 GT LoRA 相同，只是监督目标从 $G$ 变为 $T_{\mathrm{moe}}$。

\paragraph{MoE Teacher LoRA 结果。}
MoE Teacher LoRA 在 MagicBrush-dev 上得到 TO-SSIM 0.763、TO-PSNR 19.01、BG-SSIM 0.852。它的意义在于：Stage 1 可以使用 target/mask 构造高质量 teacher，但 Stage 2 将这种能力蒸馏到单 LoRA 模型中。

\subsubsection{汇总分析与展望}

当前结果可以概括为：
\begin{enumerate}
  \item \textbf{Qwen2511 本身够强，但特定任务表现不佳。}客观 target 相似度较低，非常容易改动到其他地方。
  \item \textbf{GT LoRA 证明微调有效。}TO-SSIM 从 0.450 提升到 0.696，说明 LoRA 微调 Qwen 大模型对 MagicBrush 这类特定编辑任务有明显收益。但不总能准确定位编辑区域，也会对背景产生不必要变化。
  \item \textbf{MTP LoRA 通过目标编程解决局部监督问题。}clean target 把编辑区监督和背景保持显式编码进训练目标，得到比 GT LoRA 更好的结果。且人工评估发现 MTP LoRA 的编辑更加自然，且背景保持更好。
  \item \textbf{MoE Teacher LoRA 通过多专家融合提升训练信号质量。}它利用训练期 target/mask 和专家候选构造更好的 teacher，再蒸馏到单 LoRA 中，综合指标最好。但是由于专家质量差异较大（Qwen 遥遥领先另外两个）导致融合困难，即使做了融合处理，有些专家图还是会有重影，导致微调之后推理结果也有轻微重影。
\end{enumerate}

后续可以继续改进三点：
\begin{itemize}
  \item \textbf{更强的语义 mask：}当前 MTP 和 MoE 主要依赖 dataset mask 与 source-target diff mask，后续可以接入 GroundingDINO/SAM 或 VLM planner。
  \item \textbf{更自然的局部 refinement：}MoE 融合仍主要依赖 Laplacian/feather/color harmonization，可进一步接入 BrushNet、PowerPaint 或扩散式 inpainting。
  \item \textbf{区域偏好蒸馏：}MoE 已经产生区域级 winner/loser 信息，后续可以将其转化为 preference loss，让学生学习为什么某区域应该选择某个专家。
  \item \textbf{更强的专家：}当前 Pix2Pix 和 EditAR 质量较差，后续可以替换为更强的扩散式编辑模型，或者通过检索找到更适合的专家图。
\end{itemize}

\section{Performance Optimization}

本报告的个人改进重点是编辑质量与背景保持，而非纯推理加速。不过从部署角度看，本文方法仍具有一定性能优势：LoRA 只增加少量可训练参数，推理阶段不需要多模型 ensemble；MoE-Fusion Teacher 只在训练期构造监督目标，部署时仍然是单个 Qwen LoRA 模型。因此，与直接在线运行多个专家模型相比，本文方法将多专家信息离线蒸馏到单模型中，保持了推理接口简洁。

后续性能优化可以从以下方向展开：
\begin{itemize}
  \item \textbf{低步数蒸馏：}将 Qwen 高步数输出蒸馏到低步数模型或 LoRA 中，减少扩散采样时间；
  \item \textbf{Adapter/LoRA 融合：}将 MTP 的空间稀疏约束与低步数蒸馏 LoRA 结合，在保持速度的同时提升背景稳定性；
  \item \textbf{候选缓存：}MoE teacher 构造阶段可离线缓存 P2P、Qwen、EditAR 候选，避免训练时重复推理；
  \item \textbf{评测流水线并行化：}对 MagicBrush/HQ-Edit 等 benchmark 的图像生成与指标计算进行批量化，降低实验迭代成本。
\end{itemize}

\section{Conclusion}

本文围绕 instruction-guided image editing 任务，对小组半成品报告进行了结构对齐与内容补全。小组复现实验表明：Qwen-Image-Edit 在指令执行正确性上最强，但背景保持存在短板；InstructPix2Pix/P2P 背景保持更稳定但复杂语义能力有限；EditAR 在当前设定中整体表现较弱。基于这一观察，本文在 Method Improvement 中提出面向 Qwen-Image-Edit 的监督目标改进：GT LoRA 验证微调有效性，MTP 通过 clean target 显式约束背景保持，MoE-Fusion Teacher 则利用 P2P/Qwen/EditAR 多专家候选构造区域级 teacher 并蒸馏到单一 LoRA 模型中。实验结果显示，该方法能够显著提升 MagicBrush-dev 上的目标一致性与背景保持能力。

后续工作将继续补全三方面内容：第一，完善小组成员姓名、分工与 baseline 可视化图；第二，在 HQ-Edit、Reason-Edit、GIE-Bench 等更复杂 benchmark 上验证泛化能力；第三，将本文的 MTP/MoE 质量改进与低步数蒸馏、采样调度优化结合，进一步提升推理效率。
